Turbo-Garbanzo

View Live Site →
Next.js · Supabase · PostgreSQL · Vercel

A full-stack employee scheduling and time-tracking application built as the capstone project for CIS-248 Advanced Application Development. The app supports role-based dashboards for employees, managers, and admins, with real-time data powered by Supabase and deployed on Vercel.

Features

  • Role-Based Access — Separate dashboards and permissions for employees, managers, and admins.
  • Employee Scheduling — Managers can create and manage shift schedules for their teams.
  • Time Tracking — Employees can clock in and out with time records stored in the database.
  • Supabase Backend — PostgreSQL database with row-level security policies controlling data access per role.
  • Real-Time Data — Supabase provides live data updates without requiring manual refreshes.
  • Vercel Deployment — Continuously deployed via Vercel with GitHub integration.

My Role

I served as an unofficial Scrum Master on a team of three, coordinating the team, driving project decisions, and handling a large portion of the backend work. My responsibilities included writing and managing Supabase row-level security policies, backend logic for multiple pages, and debugging across the full stack. Frontend design was primarily handled by my teammate Kay.

Tech Stack

  • Next.js — React framework with App Router for full-stack development
  • Supabase — Backend-as-a-service providing PostgreSQL, auth, and real-time data
  • PostgreSQL — Relational database with row-level security
  • JavaScript — Typed throughout for reliability
  • Vercel — Deployment and hosting
  • GitHub — Version control and team collaboration
