Scrum 3 — Study Service

View Live Site →
PHP · MySQL · RESTful · CRUD · JSON

A team-built RESTful web service for managing study groups, projects, and tasks. Built for CIS-266 using PHP and MySQL, the app provides full CRUD functionality across three related database entities — users, projects, and tasks — via a JSON-based REST API and a linked HTML client.

Features

  • User Management — Create, view, update, and delete users with roles such as Student, Tutor, Mentor, and Coordinator.
  • Project Tracking — Manage study projects with categories, statuses, due dates, and ownership tied to users.
  • Task Management — Assign tasks to users within projects, with priority levels, statuses, and due dates.
  • RESTful API — PHP-based provider handles all CRUD operations, returning data for consumption by the HTML client.
  • Relational Database — MySQL schema with foreign key constraints linking tasks to projects and users, with cascading deletes.
  • HTML Client — A simple front-end client links to all provider endpoints for viewing and managing records.

My Role

I worked as a software developer on a team of three alongside Rylee Smith and Kaleb. My primary contributions were the database routing and the create and delete functions across all three entities — users, projects, and tasks.

Tech Stack

  • PHP — Server-side REST API provider
  • MySQL — Relational database with three linked tables
  • HTML/CSS — Front-end client interface
  • mysqli — Database access layer
  • InfinityFree — Live PHP/MySQL hosting
  • GitHub — Version control and team collaboration

Live Demo

The application is hosted live at scrum3.infinityfreeapp.com. From the home page you can navigate to Users, Tasks, and Projects to view, create, update, and delete records.
