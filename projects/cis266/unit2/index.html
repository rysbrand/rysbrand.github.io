CIS-266 Unit 2 — RESTful Web Service

PHP · MySQL · RESTful · AJAX · JSON

A solo RESTful AJAX/JSON/PHP/MySQL web service built for CIS-266 Unit 2. The project consists of a PHP service provider exposing flower inventory data from a MySQL database, and an HTML/JavaScript client consuming the API via AJAX requests.

Features

  • RESTful API Provider — PHP endpoints return flower records as JSON, supporting retrieval of all records, a single record by ID, and table metadata.
  • AJAX Client — HTML/JavaScript client pages consume the API asynchronously without page reloads.
  • MySQL Database — A custom ysbr_flowers table stores flower name, color, and quantity data.
  • OOP & Procedural Versions — The provider includes both an object-oriented and a procedural implementation of the database access layer.
  • API Testers — Dedicated tester pages allow direct testing of each endpoint independently.

Sample Code

The API endpoint returning all flower records from the database:

<?php
require_once 'class_lib/studentDB_Access.php';

$db = new MurachDB_Access();
$result = $db->displayRecords("ysbr_flowers");

if ($result && mysqli_num_rows($result) > 0) {
    print("all flower records: <br />");
    while ($row = mysqli_fetch_assoc($result)) {
        print($row["flowerName"] . " " 
            . $row["flowerColor"] . " " 
            . $row["flowerAmount"] . "<br />");
    }
} else {
    print("No records found");
}
?>

Database Schema

The ysbr_flowers table stores flower inventory data:

CREATE TABLE ysbr_flowers (
  id           INT AUTO_INCREMENT PRIMARY KEY,
  flowerName   VARCHAR(250) NOT NULL,
  flowerColor  VARCHAR(250) NOT NULL,
  flowerAmount INT NOT NULL
);

Tech Stack

  • PHP — RESTful service provider
  • MySQL — Database backend
  • AJAX / JavaScript — Asynchronous client-side requests
  • HTML/CSS — Client interface
  • JSON — Data interchange format
