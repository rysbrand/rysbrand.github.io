Scrum 4 — Live Poll App

Node.js · Socket.io · Express · JavaScript

A real-time polling application built for CIS-266 Scrum 4. Users vote on their favorite programming language and results update instantly across all connected clients using WebSockets via Socket.io. Built as a two-person team project with Rylee Smith.

Features

  • Real-Time Updates — Vote results broadcast instantly to all connected clients using Socket.io WebSockets — no page refresh needed.
  • Vote Switching — Users can change their vote and the previous vote is automatically removed before the new one is counted.
  • Poll Reset — Admin can reset all votes to zero, broadcasting the cleared state to all clients.
  • Live User Tracking — Each connected socket is tracked individually to prevent duplicate votes.
  • Static File Serving — Express serves the front-end client from the public folder.

My Role

I worked on a team of two with Rylee Smith. I handled the frontend — building the UI and curating the emoji set for the poll options 🌸🍓🎀💗. Rylee handled debugging and presentation materials.

Sample Code

Vote handling with switch detection — if a user already voted, their previous vote is removed before the new one is counted:

socket.on("vote", (optionId) => {
    if (votes[optionId] !== undefined) {
        const previousVote = userVotes[socket.id];

        // If user voted before, remove old vote
        if (previousVote !== undefined) {
            votes[previousVote] -= 1;
        }

        // Add new vote
        votes[optionId] += 1;
        userVotes[socket.id] = optionId;

        // Broadcast updated results to all clients
        io.emit("poll_update", buildDeliverable());
    }
});

Poll reset broadcasts cleared state to all connected clients:

socket.on("reset_poll", () => {
    votes = Object.fromEntries(options.map((o) => [o.id, 0]));
    io.emit("poll_update", buildDeliverable());
});
